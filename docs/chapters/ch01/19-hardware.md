# 1.9 映射到硬件

C++ 提供了从语言结构到硬件的直接映射。大多数基本操作都由一两条机器指令实现。例如，`x = y + z` 通常对应一条加法指令。

C++ 的数据模型基于计算机内存的基本概念。C++ 将内存视为一系列可寻址的字节。基本类型（如 `int`、`double`）占用若干连续的字节。

## 1.9.1 赋值和初始化

赋值（assignment）和初始化（initialization）在 C++ 中有本质区别：

- **初始化**：在对象创建时赋予初始值。对于基本类型，未初始化的局部变量包含未定义的值。
- **赋值**：在对象已经存在后修改其值。

```cpp
int x = 7;      // 初始化
int y;          // 未初始化（局部变量）
y = 8;          // 赋值
```

始终初始化变量是一个好习惯：

```cpp
int y {};       // 初始化为 0
int z {42};     // 初始化为 42
```

使用 `{}` 初始化可以避免窄化转换（narrowing conversion）：

```cpp
int i1 = 7.8;   // i1 变成 7（窄化，可能产生警告）
int i2 {7.8};   // 错误：窄化转换
```

## 1.9.2 类型别名

类型别名（type alias）为类型提供一个新的名字，使代码更易读：

```cpp
using size_t = unsigned int;        // C++11 风格
typedef unsigned int size_t;        // 传统 C 风格
```

推荐使用 `using` 语法，因为它更清晰，且能更好地与模板配合。

## 1.9.3 类型转换

C++ 提供了多种类型转换方式：

```cpp
double d = 3.14;
int i = static_cast<int>(d);    // 显式转换，i == 3
```

`static_cast` 是最常用的转换，用于相关类型之间的合理转换。其他转换包括：
- `reinterpret_cast`：将一种类型按位重新解释为另一种类型
- `const_cast`：移除 `const` 限定
- `dynamic_cast`：在类层次结构中进行运行时检查的转换

应尽量避免使用 C 风格的类型转换 `(type)value`，因为它不够明确，容易隐藏错误。

## 1.9.4 对齐

对象在内存中的对齐（alignment）会影响性能和正确性。`alignof` 返回类型的对齐要求，`alignas` 可以指定对齐：

```cpp
alignas(16) char buffer[1024];  // 16 字节对齐的缓冲区
```

大多数情况下，默认对齐已经足够，不需要手动指定。
