# 3.2 分离编译

C++支持“分离编译”的机制，即用户代码只能看到所使用的类型和函数的声明部分。实现这种方式有两种途径：

- 头文件（§3.2.1）：将各种声明放在单独的文件中，这些文件就被称为头文件。在需要使用这些声明的地方，通过`#include`指令引入相应的头文件即可。
- 模块（§3.2.2）：定义模块(`module`)文件，将其单独编译，然后在需要时导入(`import`)该模块到其他地方。只有那些被明确导出的变量或函数，才能被导入该模块的代码所使用。

这两种方式都可以将程序拆分成一组相互独立的代码片段。这种划分方式有助于缩短编译时间，同时也能确保程序中各个逻辑上独立的组成部分彼此分离，从而降低出错的可能性。所谓“库”，通常指由多个单独编译而成的代码片段组成的集合，比如各种函数。

使用头文件来组织代码的做法可以追溯到C语言的早期阶段，至今仍是最为常见的编码方式。而模块化的用法则是C++20的新特性，它在提升代码的可维护性及编译效率方面有着显著的优势。

## 3.2.1 Header Files

通常，我们会将用于指定某段代码所对应接口的声明，放在一个与其用途相对应的文件中。例如：

```cpp
// Vector.h:

class Vector {
public:
    Vector(int s);
    double& operator[](int i);
    int size();
private:
    double* elem;   // elem指向一个数组，该数组承载sz个double
    int sz;
};
```

C++ 支持分离编译，允许将程序分割成多个源文件。用户代码通常放在 `.cpp` 文件中，而接口声明则放在 `.h`（头文件）中。这种分离使得我们可以独立编译各个源文件，最后再将它们链接成一个可执行程序。

### 3.2.1 分离编译的优势 {#3.2.1}

分离编译带来了几个重要优势：

- **更快的编译速度**：修改一个文件后，只需重新编译该文件及其依赖，而不必重新编译整个程序
- **逻辑组织**：将相关功能放在同一个文件中，使代码结构更清晰
- **代码复用**：同一个源文件可以被多个程序共享

传统上，C++ 使用 `#include` 指令来包含头文件。例如：

```cpp
#include <iostream>            // 包含 I/O 流库的声明

int main()
{
    std::cout << "Hello, World!\n";
}
```

`#include` 本质上是一个文本复制操作——预处理器将头文件的全部内容复制到 `#include` 指令所在的位置。这种方式简单直接，但也存在一些问题：编译速度慢、宏污染、包含顺序依赖等。

### 3.2.2 分离编译的限制 {#3.2.2}

C++20 引入了*模块*（modules）作为 `#include` 的现代替代方案。使用模块时，我们用 `import` 代替 `#include`：

```cpp
import std;              // 导入标准库模块

int main()
{
    std::cout << "Hello, World!\n";
}
```

模块的优势包括：

- 编译速度显著提升（模块只需编译一次，而非在每个翻译单元中重复编译）
- 更好的封装性（模块可以控制哪些实体对外可见）
- 消除宏污染和包含顺序问题
- 更清晰的依赖关系

不过，截至 C++20，以模块形式提供整个标准库（`import std;`）尚未成为强制标准。如果你的编译器不支持 `import std;`，可以继续使用传统的 `#include <iostream>` 方式。大多数主流编译器（GCC、Clang、MSVC）已在逐步支持模块。
