# 3.5 建议

[1] 区分声明（用作接口）与定义（用作实现）；§3.1。
[2] 在支持模块的地方，优先使用模块而非头文件；§3.2.2。
[3] 使用头文件表示接口并强调逻辑结构；§3.2；[CG: SF.3]。
[4] 在实现其函数的定义文件中 `#include` 该头文件；§3.2；[CG: SF.5]。
[5] 避免在头文件中进行非内联函数定义；§3.2；[CG: SF.2]。
[6] 使用命名空间表达逻辑结构；§3.3；[CG: SF.20]。
[7] 将 using 指令用于过渡、基础库（如 `std`）或局部作用域；§3.3；[CG: SF.6] [CG: SF.7]。
[8] 不要在头文件中使用 using 指令；§3.3；[CG: SF.7]。
[9] 按值传递“小”值，按引用传递“大”值；§3.4.1；[CG: F.16]。
[10] 优先使用 const 引用传递，而非普通引用传递；§3.4.1；[CG: F.17]。
[11] 通过函数返回值返回结果（而不是通过输出参数）；§3.4.2；[CG: F.20] [CG: F.21]。
[12] 不要过度使用返回类型推导；§3.4.2。
[13] 不要过度使用结构化绑定；具名返回类型往往使代码更易读；§3.4.5。
