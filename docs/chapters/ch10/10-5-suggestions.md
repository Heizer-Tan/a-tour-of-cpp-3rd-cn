## 10.5 建议

[1] 使用 `std::string` 来拥有字符序列；§10.2；[CG: SL.str.1]。
[2] 优先选择字符串操作而不是 C 风格的字符串函数；§10.1。
[3] 使用 `string` 声明变量和成员，而不是将其用作基类；§10.2。
[4] 按值返回字符串（依靠移动语义和拷贝省略）；§10.2, §10.2.1；[CG: F.15]。
[5] 直接或间接使用 `substr()` 读取子串，使用 `replace()` 写入子串；§10.2。
[6] 字符串可以根据需要增长和收缩；§10.2。
[7] 当需要范围检查时，使用 `at()` 而不是迭代器或 `[]`；§10.2, §10.3。
[8] 当希望优化速度时，使用迭代器和 `[]` 而不是 `at()`；§10.2, §10.3。
[9] 使用范围 `for` 安全地最小化范围检查；§10.2, §10.3。
[10] 字符串输入不会溢出；§10.2, §11.3。
[11] 仅在必要时使用 `c_str()` 或 `data()` 生成字符串的 C 风格字符串表示；§10.2。
[12] 使用 `stringstream` 或通用的值提取函数（例如 `to<X>`）进行字符串的数字转换；§11.7.3。
[13] 可以使用 `basic_string` 制作任意类型的字符的字符串；§10.2.1。
[14] 对于意图作为标准库字符串的字符串字面量，使用 `s` 后缀；§10.3；[CG: SL.str.12]。
[15] 使用 `string_view` 作为需要读取以多种方式存储的字符序列的函数的参数；§10.3；[CG: SL.str.2]。
[16] 使用 `string_span<char>` 作为需要写入以多种方式存储的字符序列的函数的参数；§10.3；[CG: SL.str.2] [CG: SL.str.11]。
[17] 将 `string_view` 视为一种带大小的指针；它不拥有其字符；§10.3。
[18] 对于意图作为标准库 `string_view` 的字符串字面量，使用 `sv` 后缀；§10.3。
[19] 对于大多数正则表达式的常规用途，使用 `regex`；§10.4。
[20] 除了最简单的模式外，优先使用原始字符串字面量来表达模式；§10.4。
[21] 使用 `regex_match()` 来匹配完整的输入；§10.4, §10.4.2。
[22] 使用 `regex_search()` 在输入流中搜索模式；§10.4.1。
[23] 正则表达式记法可以调整为匹配各种标准；§10.4.2。
[24] 默认的正则表达式记法是 ECMAScript 的；§10.4.2。
[25] 要克制；正则表达式很容易变成只写语言；§10.4.2。
[26] 注意 `\i`（其中 `i` 是数字）允许你根据先前的子模式来表达子模式；§10.4.2。
[27] 使用 `?` 使模式“懒惰”；§10.4.2。
[28] 使用 `regex_iterator` 在流上迭代查找模式；§10.4.3。
