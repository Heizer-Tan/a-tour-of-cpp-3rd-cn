# 12.9 建议

[1] STL 容器定义了一种序列；[§12.2](12-2-vector.md)。
[2] STL 容器是资源句柄；[§12.2](12-2-vector.md)，[§12.3](12-3-list.md)，[§12.5](12-5-map.md)，[§12.6](12-6-unordered-map.md)。
[3] 使用 `vector` 作为默认容器；[§12.2](12-2-vector.md)，[§12.8](12-8-container-overview.md)；[CG: SL.con.2]。
[4] 为简单遍历容器，使用范围 `for` 或一对 `begin`/`end` 迭代器；[§12.2](12-2-vector.md)，[§12.3](12-3-list.md)。
[5] 使用 `reserve()` 以避免使指向元素的指针和迭代器失效；[§12.2](12-2-vector.md)。
[6] 不要在未测量的情况下假定 `reserve()` 能带来性能收益；[§12.2](12-2-vector.md)。
[7] 在容器上使用 `push_back()` 或 `resize()`，而不是在数组上使用 `realloc()`；[§12.2](12-2-vector.md)。
[8] 不要使用指向已改变大小之后的 `vector` 的迭代器；[§12.2](12-2-vector.md)；[CG: ES.65]。
[9] 不要假定 `[]` 会做范围检查；[§12.2](12-2-vector.md)。
[10] 当你需要保证范围检查时，使用 `at()`；[§12.2](12-2-vector.md)；[CG: SL.con.3]。
[11] 使用范围 `for` 和标准库算法，以零额外成本避免范围错误；[§12.2.2](12-2-vector.md#1222-范围检查)。
[12] 元素会被拷贝进容器；[§12.2.1](12-2-vector.md#1221-元素)。
[13] 为保留元素的多态行为，应存储指针（语言内置或智能指针）；[§12.2.1](12-2-vector.md#1221-元素)。
[14] 插入操作（如 `insert()` 和 `push_back()`）在 `vector` 上往往出奇地高效；[§12.3](12-3-list.md)。
[15] 对于通常为空的很短序列，可使用 `forward_list`；[§12.8](12-8-container-overview.md)。
[16] 涉及性能时不要轻信直觉：要测量；[§12.2](12-2-vector.md)。
[17] `map` 通常实现为红黑树；[§12.5](12-5-map.md)。
[18] `unordered_map` 是哈希表；[§12.6](12-6-unordered-map.md)。
[19] 以引用传递容器，以值返回容器；[§12.2](12-2-vector.md)。
[20] 对容器用 `()` 初始化语法表示大小，用 `{}` 表示元素序列；[§5.2.3](../ch05/5-2-concrete-types.md#523-初始化容器)，[§12.2](12-2-vector.md)。
[21] 优先选择紧凑、连续的数据结构；[§12.3](12-3-list.md)。
[22] 遍历 `list` 相对昂贵；[§12.3](12-3-list.md)。
[23] 若要对大量数据快速查找，使用无序容器；[§12.6](12-6-unordered-map.md)。
[24] 若需要按顺序遍历元素，使用有序容器（如 `map` 和 `set`）；[§12.5](12-5-map.md)。
[25] 对没有自然顺序（即没有合理的 `<`）的元素类型，使用无序容器（如 `unordered_map`）；[§12.5](12-5-map.md)。
[26] 当需要在容器大小变化时保持指向元素的指针稳定时，使用关联容器（如 `map` 和 `list`）；[§12.8](12-8-container-overview.md)。
[27] 通过实验确认哈希函数是否可接受；[§12.6](12-6-unordered-map.md)。
[28] 用按位异或（`^`）组合各元素的标准哈希函数，这样得到的哈希函数常常效果不错；[§12.6](12-6-unordered-map.md)。
[29] 熟悉标准库容器，并优先使用它们而不是手写的数据结构；[§12.8](12-8-container-overview.md)。
[30] 若应用程序因内存而出现性能问题，尽量减少自由存储的使用，并/或考虑专用分配器；[§12.7](12-7-allocators.md)。
