# 前言

C++ 感觉像是一门新语言。也就是说，与在 C++98 或 C++11 中相比，今天我能更清晰、更简单、更直接地表达我的想法。此外，生成的程序能通过编译器得到更好的检查，且运行速度更快。

本书概述了由 C++20（当前的 ISO C++ 标准）所定义的 C++，以及由各大 C++ 供应商所实现的版本。此外，书中还提到了几个目前已在使用、但预计要到 C++23 才会纳入标准的库组件。

与其他现代语言一样，C++ 非常庞大，高效使用它需要大量的库。这本薄书旨在让有经验的程序员了解什么是"现代 C++"。它涵盖了大部分主要的语言特性和主要的标准库组件。本书只需一两天即可读完，但显然，编写优秀的 C++ 代码所涉及的内容远非这段时间内能学完的。然而，这里的目的不是为了精通，而是提供一个概览，给出关键示例，并帮助程序员入门。

我们的假设是你之前有过编程经验。如果没有，在继续阅读本书之前，请考虑先阅读一本教科书，例如《C++ 程序设计：原理与实践（第二版）》[Stroustrup, 2014]。即使你以前写过程序，你所使用的语言或编写的应用程序可能与此处介绍的 C++ 风格截然不同。

可以把它想象成一次城市观光游，比如哥本哈根或纽约。在短短几个小时内，你会快速浏览主要的景点，听几个背景故事，并获得一些关于下一步该做什么的建议。在这样的旅行之后，你并不算了解这座城市。你并不能理解你所看到和听到的一切；有些故事听起来可能很奇怪，甚至不可信。你不知道如何应对管理城市生活的正式和非正式规则。要真正了解一座城市，你必须住在那里，通常要住上很多年。然而，运气好的话，你会获得一点概览，对这座城市的特别之处有所了解，并明白自己可能对什么感兴趣。观光结束后，真正的探索才刚刚开始。

这次"导程"介绍了支持编程风格（如面向对象编程和泛型编程）的主要 C++ 语言特性。它并不试图提供一份详尽的、手册式的、逐一特性介绍的语言视图。按照教科书的优良传统，我尽量在尝试使用某个特性之前先对其进行解释，但这并不总是可行，而且并非每个人都会严格按顺序阅读。我假设我的读者在技术上具有一定的成熟度，因此，鼓励读者利用书中的交叉引用和索引。

同样，这次导程通过示例而非详尽列举的方式来介绍标准库。鼓励读者根据需要寻找额外的补充材料。C++ 的生态系统远不止 ISO 标准所提供的设施（例如库、构建系统、分析工具和开发环境）。网上有海量的材料（质量参差不齐）。大多数读者会从 CppCon 和 Meeting C++ 等会议中找到有用的教程和概览视频。对于 ISO C++ 标准提供的语言和库的技术细节，我推荐 [Cppreference]。例如，当我提到一个标准库函数或类时，可以很容易地查到它的定义，通过查阅文档，还可以发现许多相关的设施。

这次导程将 C++ 作为一个整体来呈现，而不是像"层级蛋糕"那样分层展示。因此，我很少指明某个语言特性是出现在 C、C++98 还是后来的 ISO 标准中。这类信息可以在第 19 章（历史与兼容性）中找到。我专注于基础知识并力求简洁，但我并没有完全抵制住"过度展现"新特性的诱惑，如模块 (§3.2.2)、概念 (§8.2) 和协程 (§18.6)。略微偏向近年的发展，似乎也能满足许多已经了解某些旧版本 C++ 的读者的好奇心。

编程语言参考手册或标准仅说明"可以做什么"，但程序员通常更感兴趣的是学习"如何用好这门语言"。这一方面部分体现在主题的选择上，部分体现在正文中，特别是"建议（advice）"章节中。关于什么是优秀的现代 C++ 的更多建议，可以在《C++ 核心指南》（C++ Core Guidelines）[Stroustrup, 2015] 中找到。核心指南可以作为进一步探索本书所述思想的良好来源。你可能会注意到，核心指南与本书在建议的表述形式甚至编号上有着显著的相似性。原因之一是，《C++ 导程》的第一版曾是最初核心指南的主要来源。

### 读者

本书的目标读者是：

- 想要了解现代 C++ 的程序员；
- 需要快速了解 C++ 关键特性的软件工程师；
- 想要更新 C++ 知识的有经验的程序员；
- 作为 C++ 课程的辅助教材；
- 作为 C++ 编程的快速参考。
