# 前言

> 欲授人业，辞简为先。
>
> —— 西塞罗

C++ 感觉像一门新的语言。也就是说，与 C++98 或 C++11 相比，如今我能更清晰、更简单、更直接地表达我的想法。此外，生成的程序能更好地通过编译器检查，并且运行速度也更快。

本书概述了由 C++20（当前的 ISO C++ 标准）所定义的 C++，以及由各大 C++ 供应商所实现的版本。此外，书中还提到了几个目前已在使用、但预计要到 C++23 才会纳入标准的库组件。

和其他现代语言一样，C++ 是一门庞大的语言，要进行有效的使用还需要掌握大量的库。这本薄书旨在让有经验的程序员了解什么是"现代 C++"。它涵盖了大部分主要的语言特性和主要的标准库组件。本书只需一两天即可读完，但显然，编写优秀的 C++ 代码所涉及的内容远非这段时间内能学完的。然而，这里不求掌握，只求概述、举例，并帮程序员顺利上手。

我们假设你之前有过编程经验。如果没有，在继续阅读本书之前，请考虑先阅读一本教科书，例如《C++ 程序设计：原理与实践（第二版）》[Stroustrup, 2014][^1]。即使你以前写过程序，你所使用的语言或编写的应用程序可能与此处介绍的 C++ 风格截然不同。

把这想象成一次城市观光之旅，比如游览哥本哈根或纽约。短短几个小时里，你快速扫一眼主要景点，听几句背景故事，再得到一些关于接下来该做什么的建议。但一趟游览下来，你并不真正了解这座城市。你无法完全理解所看所闻的一切；有些故事可能听起来奇怪，甚至不太可信。你也不知道如何遵循这座城市中那些或明或暗的规则。要真正认识一座城市，你得住下来，往往要住上好几年。不过，运气好的话，你会获得一点整体印象，知道这座城市的特别之处，也能对哪些地方可能感兴趣有个概念。游览结束后，真正的探索才刚刚开始。

这次“游览”将介绍 C++ 的主要语言特性，并说明它们如何支持面向对象编程和泛型编程等编程风格。本书并不打算按参考手册的方式，逐个特性地详细讲解这门语言。按照优秀教材的传统，我会尽量在使用某个特性之前先加以解释，但这并非总能如愿，而且也不是所有人都会严格按顺序阅读。我假设读者已经具备一定的技术成熟度。因此，建议读者善用书中的交叉引用和索引。

同样，本次“游览”也会通过实例来介绍标准库，而非面面俱到。建议读者根据需要自行查找补充材料。C++ 的生态远不止 ISO 标准提供的那些设施（例如库、构建系统、分析工具、开发环境）。网络上有着海量的资料，质量参差不齐。大多数读者可以在 CppCon 和 Meeting C++ 等会议上找到有用的教程和概览视频。关于 ISO C++ 标准所提供语言和库的技术细节，我推荐 [Cppreference][^2]。举个例子，当我提到某个标准库函数或类时，读者可以轻松查到它的定义，并通过阅读其文档发现许多相关的功能。

本次“游览”将 C++ 视为一个有机整体，而非层层堆叠的“千层糕”。因此，我很少特意指出某个语言特性是源自 C、C++98 还是后续的 ISO 标准。这类信息可以在第 19 章（历史与兼容性）中找到。我专注于基础知识并力求简洁，但也没能完全抵挡住对某些新特性稍加偏重的诱惑，比如模块（§3.2.2）、概念（§8.2）和协程（§18.6）。适当向近期发展的特性倾斜，似乎也能满足许多已经掌握某个旧版 C++ 的读者的好奇心。

编程语言的参考手册或标准只会告诉你“能做什么”，而程序员往往更感兴趣的是“如何用好这门语言”。本书在选题、正文以及专门的“建议”章节中，都部分回应了这一问题。关于什么样的代码才算得上优秀的现代 C++，C++ 核心指南（C++ Core Guidelines）[Stroustrup, 2015][^3] 提供了更丰富的建议。核心指南可以作为进一步探索本书所介绍思想的良好起点。你可能会注意到，核心指南与本书中的建议在表述方式甚至编号上都高度相似。原因之一是，第一版《C++ 之旅》正是核心指南早期版本的主要素材来源。
